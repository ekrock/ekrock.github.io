## [ElectroMiser Home Solar Energy Optimization Agent](https://electromiser.com/login?u=demo&p=Solar%24avez987%23%24)
Try out the [demo version](https://electromiser.com/login?u=demo&p=Solar%24avez987%23%24)! Built agent that enables homeowners with solar array, home battery, and Level 2 EV charger to understand the behavior of their solar array and Powerwall 3 home battery and home electrical usage patterns and to determine what time of day they should charge their EV at using their Level 2 EV charger and whether they need to schedule other loads (HVAC, dryer) if they want to avoid importing power from the grid. Agent prepopulates InfluxDB database with historical solar system data sourced from Powerwall and weather data for user latlong from Open Mateo. User can ask natural language questions of the agent in voice or text and get specific answers leveraging stored data. Agent also produces charts at various levels of granularity of solar system performance metrics. Agent was implemented using Claude Code in Python, stored in GitHub, and hosted on AWS. Uses: pypowerwall library to retrieve data from Tesla Owner API; Anthropic Claude Sonnet 4.6 — primary RAG generation (Q&A, daily summaries, EV charging advisor); OpenAI Whisper-1 — transcribes voice queries; OpenAI TTS-1 (alloy voice) — speaks the answer back for hands-free use.; InfluxDB Cloud — time-series storage; Open-Meteo — GTI solar forecast and cloud-cover data; ASHRAE photovoltaic production estimation algorithm to estimate expected production over the course of the year (theoretical open skies, and derated for expected monthly cloud coverage).

## [Simulation of Robotic Back End Assembly Line]([https://backendsim.com](https://backendsim.com/login?u=demo&p=CF*.iD!8.rFBruzD8W-R)
Try out the [demo version](https://backendsim.com/login?u=demo&p=CF*.iD!8.rFBruzD8W-R)! Built simulation for modeling throughput and performance for a single robotic back end assembly line for electronics products. User can download a CSV file to set up the simulation parameters, upload the edited CSV simulation configuration file, run the simulation, and see the results. Can define types of robots and define each type's capabilities and vary number of stations, parts to build, target ticks/unit, distance to central store, number of robots of each defined type, and input/output buffer sizes. ## ["Debunk That Post the Most" Custom GPT](https://chatgpt.com/g/g-693cb7d6ffd08191bc8e8d3f2b4b88ba-debunk-that-post-the-most)
Used Chat_GPT 5.2 to write custom GPT that helps you debunk long social media posts (e.g. Gish Gallops on Facebook) that contain multiple false statements. Paste in thee text of a post found on social media. It will identify up to 10 false claims and write a reply calmly debunking each one, citing credible sources. ## Resilient Supply Agent
Agentic AI solution for managing supply chain risk developed as team project using Lovable, Python, n8n, OpenAI, publicly-available risk-information databases. Wrote Watchman Agent that implemented back-end database and APIs for multi-agent coordination and monitored product delivery commitments, BoMs, and inventory on hand to detect when inventory was running low. Also wrote simulator to simulate day-by-day manufacturing activity. Risk Scanner Agent scanned public databases of risk information to identify when changes in risk environment put supplier commitments at risk. Sourcer Agent evaluated alternative suppliers for their risk and recommended optimal supplier for next shipment. Earned capstone project certification from NextGen Product Manager.
